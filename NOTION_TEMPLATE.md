## 🆕 [DB 4] 추천 종목 — 노션에서 새로 만들 DB 구조

노션 페이지 안에 **인라인 데이터베이스**를 새로 추가하고 아래 컬럼을 생성하세요.

| 컬럼명 | 속성 타입 | 설명 | 예시 |
|--------|---------|------|------|
| 날짜 | **제목(Title)** ⭐ | 날짜 텍스트, 행 구분 키 | `2026-02-19` |
| 종목명 | 텍스트(Text) | 한글 종목명 | `삼성전자` |
| 추천등급 | **선택(Select)** | A+ / A / B+ / B / C | `A+` |
| 투자기간 | **선택(Select)** | 단기 / 중기 / 장기 | `중기` |
| 추천이유 | 텍스트(Text) | 자유 텍스트 (2~3문장) | `HBM 전환 가속...` |

### 입력 예시 (하루 3~5행)

| 날짜 (Title) | 종목명 | 추천등급 | 투자기간 | 추천이유 |
|-------------|--------|---------|---------|---------|
| 2026-02-19 | 삼성전자 | A+ | 중기 | HBM 전환 가속 및 파운드리 회복. 저점 분할매수 구간. |
| 2026-02-19 | SK하이닉스 | A | 단기 | HBM3E 독점 공급 지위. 고수익성 구조 지속. |
| 2026-02-19 | NAVER | B+ | 장기 | B2B AI 전환 가속. 수익화 초입 구간. |

> ⭐ **날짜 컬럼이 반드시 Title(제목) 타입**이어야 합니다.  
> 노션에서 DB를 새로 만들면 기본으로 생성되는 "이름" 컬럼을 **"날짜"로 이름 변경**해서 사용하세요.  
> Text 타입으로 만들면 날짜 필터가 작동하지 않습니다.

## ✏️ 작성 규칙

### 섹션 헤더
- 헤더는 반드시 heading_1(#) 또는 heading_2(##) 형식으로 작성
- 헤더 텍스트는 정확히 일치해야 합니다: 시장 요약 / 주도 섹터 / 특이 종목 / 주요 뉴스 / 추천 종목

### 데이터 구분자
- 파이프(|)로 항목을 구분합니다
- 앞뒤 공백은 자동으로 처리되므로 " | " 또는 "|" 모두 가능

### 섹터 등락률
- 양수: +3.42% (+ 기호 포함)
- 음수: -1.43% (- 기호 포함)

### 특이 종목
- 형식: 종목명 | 이유 | 현재가(원) | 등락률(%)
- 이유는 15자 이내 간결하게

### 추천 종목 등급
- A+ / A / B+ / B / C 중 하나
- 첫 번째 항목이 자동으로 "TOP PICK"으로 표시됨

## 🔧 GitHub Secrets 설정 방법

저장소 → Settings → Secrets and variables → Actions → New repository secret

| 이름 | 값 |
|------|-----|
| NOTION_TOKEN | secret_xxxxxxxxxx (보유 중인 API 키) |
| NOTION_PAGE_ID | 노션 페이지 URL의 마지막 32자리 |

### 페이지 ID 찾는 법
노션 페이지 URL:
https://www.notion.so/마켓-인사이트-20250218-**a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4**

굵게 표시된 32자리가 페이지 ID입니다.
(하이픈 포함 형식: a1b2c3d4-e5f6-a1b2-c3d4-e5f6a1b2c3d4)

## 📂 GitHub Pages 배포 순서

1. GitHub에 새 저장소 생성 (예: market-dashboard)
2. 아래 파일들을 업로드:
   - dashboard.html  ← 메인 페이지
   - fetch_data.py   ← 데이터 수집 스크립트
   - .github/workflows/update.yml  ← 자동화 워크플로우
3. Settings → Pages → Source: Deploy from a branch → main 브랜치 루트(/)
4. Settings → Secrets에 NOTION_TOKEN, NOTION_PAGE_ID 추가
5. Actions 탭에서 워크플로우 수동 실행 (첫 data.json 생성)
6. 이후 매일 오전 8시 자동 실행

배포 URL: https://[username].github.io/[repository-name]/dashboard.html
